
# Phase 1 実装仕様書・実装指示書

## 📋 Phase 1 概要

**目的**: calendar-for-mywork を「見るだけツール」から「管理可能なツール」へ進化させる

**対象機能**:
1. イベント編集ダイアログ
2. ビジネスアワー設定
3. イベント色の自動割り当て

**期間**: 3週間（Week 1-3）

**完了条件**: Google Calendar の基本的な使用フローが calendar-for-mywork で完結できること

# Phase 1 実装仕様書（ドラフト版）

## 機能 1: イベント編集ダイアログ

### 概要
カレンダー上のイベントをクリックまたはダブルクリックすると、詳細編集ダイアログが表示される。
ユーザーはここでイベントの情報を確認・編集・削除できる。

### 起動パターン
- **クリック**: イベントをシングルクリック → ダイアログ表示
- **ダブルクリック**: イベントをダブルクリック → ダイアログ表示
- **コンテキストメニュー**: イベント右クリック → 「編集」選択 → ダイアログ表示

### UI 構成

```
┌─────────────────────────────────────────────────┐
│  イベント編集                            [×] 閉じる │
├─────────────────────────────────────────────────┤
│                                                   │
│  【基本情報】                                      │
│  ┌──────────────────────────────────────────┐   │
│  │ タイトル                                  │   │
│  │ [入力フィールド]                          │   │
│  │                                            │   │
│  │ 開始日時 *            終了日時 *          │   │
│  │ [日時選択器]        [日時選択器]          │   │
│  │                                            │   │
│  │ ☐ 全日イベント                           │   │
│  │                                            │   │
│  │ 説明/メモ                                  │   │
│  │ [テキストエリア × 3行]                    │   │
│  │                                            │   │
│  │ 場所                                       │   │
│  │ [入力フィールド]                          │   │
│  │                                            │   │
│  └──────────────────────────────────────────┘   │
│                                                   │
├─────────────────────────────────────────────────┤
│ [キャンセル] [削除] [保存]                        │
└─────────────────────────────────────────────────┘
```

### 編集可能フィールド（初版）

| フィールド | 必須 | 入力方式 | バリデーション |
|-----------|------|--------|---------------|
| タイトル | ✅ | テキスト | 1字以上 100字以下 |
| 開始日時 | ✅ | 日時選択器 | 終了日時より前 |
| 終了日時 | ✅ | 日時選択器 | 開始日時より後 |
| 全日イベント | ❌ | チェックボックス | - |
| 説明/メモ | ❌ | テキストエリア | 1000字以下 |
| 場所 | ❌ | テキスト | 100字以下 |

### ボタン動作

| ボタン | 動作 | 確認 |
|--------|------|------|
| **キャンセル** | ダイアログを閉じて変更を破棄 | なし |
| **削除** | イベントを削除する | **あり**（「本当に削除?」） |
| **保存** | 変更を保存してダイアログを閉じる | バリデーション後 |

### バリデーションルール
- タイトルが空白ならエラー：「タイトルを入力してください」
- 開始日時 ≥ 終了日時 ならエラー：「終了時刻は開始時刻より後にしてください」
- その他は入力値を記録

### 削除時の確認ダイアログ
```
┌─────────────────────────────────┐
│ イベントを削除します              │
│                                   │
│ 「○○○○」                         │
│ ▓▓▓▓年▓月▓日 ▓:▓▓ - ▓:▓▓      │
│                                   │
│ この操作は取り消せません。         │
│ 本当に削除しますか？             │
│                                   │
│ [キャンセル] [削除]               │
└─────────────────────────────────┘
```

### データ永続化
- 保存時にバックアンド API に POST/PUT
- エラー時はトースト通知「保存に失敗しました」

## 機能 2: ビジネスアワー設定

### 概要
営業時間帯を視覚的に強調表示する機能。営業外の時間帯を薄い灰色で背景に表示することで、
ユーザーが「営業時間内だけを確認すればいい」と判断できるようにする。

### 設定方法

#### グローバル設定（/settings ページ）
```
┌────────────────────────────────────┐
│ カレンダー設定                       │
├────────────────────────────────────┤
│                                     │
│ 【営業時間設定】                    │
│                                     │
│ ☑ 営業時間を表示する                │
│                                     │
│ 月曜日 ☑ 09:00 - 17:30            │
│ 火曜日 ☑ 09:00 - 17:30            │
│ 水曜日 ☑ 09:00 - 17:30            │
│ 木曜日 ☑ 09:00 - 17:30            │
│ 金曜日 ☑ 09:00 - 17:30            │
│ 土曜日 ☐ (無効)                     │
│ 日曜日 ☐ (無効)                     │
│                                     │
│ [リセット] [保存]                   │
└────────────────────────────────────┘
```

### データ構造
```typescript
interface BusinessHours {
  enabled: boolean;
  weekDays: {
    monday: { enabled: boolean; startTime: "09:00"; endTime: "17:30" }
    tuesday: { enabled: boolean; startTime: "09:00"; endTime: "17:30" }
    // ... 以下同じ
  }
}

// 例
const defaultBusinessHours: BusinessHours = {
  enabled: true,
  weekDays: {
    monday: { enabled: true, startTime: "09:00", endTime: "17:30" },
    tuesday: { enabled: true, startTime: "09:00", endTime: "17:30" },
    wednesday: { enabled: true, startTime: "09:00", endTime: "17:30" },
    thursday: { enabled: true, startTime: "09:00", endTime: "17:30" },
    friday: { enabled: true, startTime: "09:00", endTime: "17:30" },
    saturday: { enabled: false, startTime: "00:00", endTime: "00:00" },
    sunday: { enabled: false, startTime: "00:00", endTime: "00:00" }
  }
}
```

- 営業外時間帯では「イベント作成」は後日オプション追加時に制限可能

## 機能 3: イベント色の自動割り当て

### 概要
イベントに「カテゴリ」を付与すると、その カテゴリに応じた色が自動的に割り当てられる。
ユーザーは色を手動で選ぶ必要がなく、統一的な色体系でイベントが表示される。

### カテゴリ ＆ デフォルトカラーマップ

```typescript
enum EventCategory {
  MEETING = 'meeting',
  DEADLINE = 'deadline',
  PERSONAL = 'personal',
  BREAK = 'break',
  OTHER = 'other'
}

const defaultCategoryColors: Record<EventCategory, string> = {
  'meeting': '#4285f4',    // 青
  'deadline': '#ea4335',   // 赤
  'personal': '#34a853',   // 緑
  'break': '#fbbc04',      // 黄
  'other': '#9c27b0'       // 紫
};
```

### UI での カテゴリ選択

#### (1) イベント作成時
```
【新規イベント】
タイトル: [入力]
開始: [日時] 終了: [日時]
カテゴリ: 
  ◯ 会議（青）
  ◯ 期限（赤）
  ◯ 個人（緑）
  ◯ 休憩（黄）
  ◯ その他（紫）
[作成]
```

#### (2) イベント編集ダイアログ内
```
【詳細】
...（タイトル、日時など）
カテゴリ:
  [ドロップダウン v]
  - 会議
  - 期限
  - 個人
  - 休憩
  - その他
```

### データモデルの拡張
```typescript
interface CalendarItem {
  id: string;
  name: string;
  description: string;
  startTime: DateTime;
  endTime: DateTime;
  
  // 【追加フィールド】
  category?: EventCategory;  // ← 新規
  color?: string;            // ← 上書き用（手動色指定）
  
  // （既存フィールド）
  ...
}
```

### 色の決定ロジック
```typescript
function getEventColor(item: CalendarItem): string {
  // (1) 手動色指定があれば優先
  if (item.color) {
    return item.color;
  }
  
  // (2) カテゴリから色を取得
  if (item.category && defaultCategoryColors[item.category]) {
    return defaultCategoryColors[item.category];
  }
  
  // (3) デフォルト色
  return '#4285f4';  // 青
}
```

### 表示時の適用
- Week View のイベントブロック背景色に適用
- Month View のイベント背景色に適用
- サイドバーの凡例に表示（「会議：青」など）

### ユーザーが色をカスタマイズしたい場合（将来）
```
【色カスタマイズ】（Phase 2 以降）
会議:    [████][色選択...]
期限:    [████][色選択...]
個人:    [████][色選択...]
...
```
